# ZenixUI World Specification Template

Every new world in ZenixUI must begin by defining this specification. 
Design and engineering do not begin until this document is approved. 

---

## 1. Core Identity
- **World Name:** (e.g., Aurora, Hanami)
- **Internal ID:** (e.g., World 01, World 02)
- **Theme/Category:** (e.g., Technology, Hospitality)
- **One-Sentence Description:** (If they answer "It's the X one," we fail. It must evoke a feeling.)

## 2. Emotion
- **Primary Emotion:** (e.g., Confidence, Peace, Creative Freedom)
- **Secondary Emotion:** (e.g., Intelligence, Grace)

## 3. Typography
- **Heading Font:** (e.g., Inter, Playfair Display)
- **Body Font:** (e.g., Inter)
- **Setting Rules:** (e.g., "Headings must be tracked tightly", "Wide letter-spacing for relaxation")

## 4. Behavior & Motion
- **Motion Philosophy:** (e.g., Purposeful Motion, Motion Waits)
- **Timing:** (e.g., Fast/Snappy, Slow/Deliberate)

## 7. Sound (Future)
- **Ambient Tone:** (e.g., Deep space hum, Nature whispers)

## 8. Accessibility & Responsiveness
- **Reduced Motion Strategy:** (e.g., Fall back to simple opacity fades)
- **Contrast:** (e.g., Strict high contrast, soft pastels with pure black text)

## 9. Recognition & Memory
- **The 3-Second Recognition Test:** What are the immediate visual cues?
- **The 30-Second Memory Test:** What 3 adjectives should a user remember after 30 seconds?
